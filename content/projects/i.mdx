# The Mission

In closed enclosures like acrylic boxes, heat often becomes trapped, causing temperatures to spike beyond safe limits. Simply adding a heater isn't enough; you need a dynamic system that can both generate and evacuate heat to maintain a delicate equilibrium.

I engineered this **Thermal Chamber** to solve the problem of thermal instability. My objective was to create a stable environment using a PTC heater and an exhaust fan controlled by a single intelligent loop, ensuring the environment remains within a precise "Dead Zone" suitable for incubation.

## The Intelligence: Hysteresis Control Logic

Instead of a simple ON/OFF thermostat that causes rapid, damaging relay switching (chattering), I implemented a **Hysteresis Algorithm**. This approach uses the system's history to prevent unnecessary oscillations and extend the lifespan of mechanical components.

### The Logic Flow:
* **Heating Phase:** If T Less Then 26.5¬∞C, the PTC Heater activates and the Fan remains OFF.
* **The Dead Zone (Stability):** Between **26.5¬∞C and 27.5¬∞C**, the system enters a "Dead Zone" where all actuators are OFF to save energy.
* **Cooling Phase:** If T More than 27.5¬∞C, the Fan activates to evacuate excess heat.

## Hardware & Architecture

The system utilizes an **ESP32** as the central brain, processing real-time sensor data while simultaneously hosting a web server for remote interaction.

| Component | Specification | Role |
| :--- | :--- | :--- |
| **Microcontroller** | ESP32 DevKit V1 | Main Processor & Wi-Fi Web Server |
| **Sensor** | DHT-11 | Temperature & Humidity Sensing |
| **Heating** | PTC Heater 12V | Thermal Generation |
| **Cooling** | Brushless DC Fan 12V | Heat Evacuation |
| **Power** | LM2596 Step-Down | Voltage Regulation (12V to 5V) |
| **Display** | LCD I2C 16x2 | Local Real-time Monitoring |

## Challenges & Engineering Evaluation

Professional engineering requires the courage to analyze failures. During the final demonstration, the exhaust fan failed to rotate despite the system status showing "ON".

**My Post-Mortem Investigation:**
1. **Mechanical Integrity:** The use of jumper wires on the relay output had low resistance to vibration. When the device was moved, the connection became loose.
2. **Contact Resistance:** I identified high contact resistance in the fuse holder, causing momentary power instability.
3. **The Solution:** For industrial-grade reliability, I concluded that moving from prototypes to **permanent terminal blocks and custom PCBs** is a non-negotiable requirement.
